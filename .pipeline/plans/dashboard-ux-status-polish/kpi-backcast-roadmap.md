# KPIバックキャスト

## 到達状態

4つのUX改善がそれぞれ単独で動くだけでなく、既存の会議一覧・詳細・辞書・声紋導線を壊さず同時に提供される。

## 品質KPI

| KPI | 目標 | 証跡 |
|---|---|---|
| 辞書状態識別 | 無効行に背景・文字の両方の無効スタイルがある | Dashboard UIテスト |
| ナビゲーション | 日本語サイドパネルが「会議一覧」 | コピー単体テスト |
| 声紋操作安全性 | 停止から音声読込完了まで操作不能、完了・失敗・10秒タイムアウトで解除 | 声紋UIテスト |
| 再文字起こし状態 | queued/runningは処理中、succeededは完了、failedは再処理失敗 | ステータス単体テスト |
| 詳細状態追従 | POST応答を即時反映し、処理中だけ2.5秒更新して旧completedへの逆戻りを防ぐ | 再処理UIテスト |
| 一覧状態追従 | 処理中だけ2.5秒サイレント更新し、完了・失敗・アンマウントで停止 | 再処理UIテスト |
| 既存回帰 | Dashboard全テスト・build成功 | コマンド証跡 |

## チェックポイント

1. 表示変更3件を対象テストで固定する。
2. 再文字起こし状態をAPI軽量データ、共通表示、詳細画面更新へ通す。
3. 全体テスト・build・GitNexusで横断回帰を確認する。

# 基本設計書：カボス日本語ダッシュボード

**バージョン**: 2.0.0
**作成日**: 2026年06月24日
**最終更新日**: 2026年07月03日

## 1. 結論

本プロダクトは日本語限定・カボス単一ブランドとする。画面上の既定ブランド、既定ボット名、既定ロケールはすべて「カボス」/日本語へ統一する。Vexaは内部API名、DB名、既存パッケージ名、互換性維持が必要な技術識別子としてのみ残す。

Docker Composeでは通常の `dashboard` もカボス既定値で起動する。`kabosu-dashboard` は共有認証や別ポート検証を行うための互換プロファイルとして残すが、二枚看板の本番運用を意味しない。

## 2. スコープ

| 区分 | 内容 |
| :-- | :-- |
| 対象 | ダッシュボードのブランド名、主要ナビゲーション、ログイン導線、会議一覧、会議参加モーダルの日本語表示 |
| 対象 | 通常 `dashboard` のカボス既定値、必要時の `kabosu-dashboard` 別起動 |
| 対象外 | API名、DB名、内部パッケージ名、既存Vexa公開ドキュメントの全面翻訳 |
| 対象外 | 独自カボスロゴ画像の制作。ロゴ未指定時は文字ロゴを表示する |

## 3. システム構成

| 画面 | 役割 | カボス版での差分 |
| :-- | :-- | :-- |
| ログイン画面 | 会議URL入力、ログイン開始 | ブランド表示と主要文言を日本語化 |
| 会議一覧 | 文字起こし済み/実行中の会議一覧 | 見出し、検索、状態フィルタ、空状態を日本語化 |
| 会議参加モーダル | 会議URL、ボット名、言語、録音/文字起こし設定 | 既定ボット名を「カボス」にし、主要操作文言を日本語化 |
| ヘッダー/サイドバー | 主要ナビゲーション | ブランド名、メニュー名、管理メニューを日本語化 |

## 4. 設定

| 項目 | 既定値 | 備考 |
| :-- | :-- | :-- |
| `DASHBOARD_BRAND_NAME` | `カボス` | 画面上の正式ブランド名 |
| `DASHBOARD_BRAND_SHORT_NAME` | `カボス` | ヘッダーやフッターの短縮名 |
| `DASHBOARD_BRAND_SLUG` | `kabosu` | ブランド識別子。技術的なCookie名とは独立 |
| `DASHBOARD_LOCALE` | `ja` | 日本語限定方針の既定 |
| `DEFAULT_BOT_NAME` | `カボス` | 会議参加時の既定ボット名 |
| `DASHBOARD_SHARED_AUTH_EMAIL` | 運用環境で必ず変更 | compose例では `tech@bonginkan.ai` を開発用既定として持つ |
| `KABOSU_DASHBOARD_HOST_PORT` | `3002` | 互換プロファイル `kabosu-dashboard` の公開ポート |

Composeでの起動例:

```bash
docker compose --profile kabosu up kabosu-dashboard
```

## 5. 重要機能の契約定義

### F-01 英語/Vexa既定Dashboardの維持（廃止）

1. **目的**: 旧方針ではVexa Dashboardを既存ユーザー向けに残す想定だった。
2. **実行者と権限**: Dashboard利用者。
3. **入力**: 通常のDashboard URL。
4. **出力**: 廃止。通常URLでもカボスブランド・日本語UIを表示する。
5. **前提条件**: なし。
6. **正常系**: `DASHBOARD_BRAND_*` 未指定でも `/api/config` の `brand.name` は `カボス`、`brand.locale` は `ja` になる。
7. **例外系**: `VEXA_API_URL` 未設定時は既存どおり設定エラーを返す。
8. **データ更新点**: なし。
9. **受入条件**:
   - `DASHBOARD_BRAND_NAME` 未指定で `/api/config` の `brand.slug` が `kabosu` になる。
   - `DASHBOARD_BRAND_NAME` 未指定で `brand.locale` が `ja` になる。

### F-02 カボス版Dashboardの表示

1. **目的**: 同じ機能をカボスブランドの日本語UIとして提供する。
2. **実行者と権限**: Dashboard利用者。
3. **入力**: カボス版Dashboard URL。
4. **出力**: カボスブランド、日本語主要導線、既定ボット名「カボス」。
5. **前提条件**: 既定値として `DASHBOARD_BRAND_NAME=カボス`、`DASHBOARD_LOCALE=ja`、`DEFAULT_BOT_NAME=カボス` 相当が有効であること。
6. **正常系**: ヘッダー、サイドバー、ログイン、会議一覧、会議参加モーダルが日本語表示になる。
7. **例外系**: 独自ロゴ未指定時は、Vexaロゴではなくブランド頭文字の文字ロゴを表示する。
8. **データ更新点**: 会議参加時の `bot_name` が未入力なら `カボス` になる。
9. **受入条件**:
   - `/api/config` の `brand.name` が `カボス` である。
   - 会議参加モーダルのボット名初期値が `カボス` である。
   - サイドバーに `会議`、`会議に参加`、`管理` が表示される。

### F-03 セッションとローカル保存の扱い

1. **目的**: 通常Dashboardと互換プロファイルを同一ブラウザで使っても、認証情報やボット名保存が実害なく扱えるようにする。
2. **実行者と権限**: Dashboard利用者。
3. **入力**: 通常Dashboardと `kabosu-dashboard` の各ログイン状態。
4. **出力**: カボス版は `kabosu-token-compose` と `kabosu-user-info-compose` を使う。
5. **前提条件**: Composeの `kabosu-dashboard` を使用すること。
6. **正常系**: 通常Dashboardと `kabosu-dashboard` のCookie名が異なる。
7. **例外系**: 運用者が同じCookie名を手動指定した場合、セッションが共有される可能性がある。
8. **データ更新点**: ブラウザCookie、localStorage。
9. **受入条件**:
   - Compose設定で `dashboard` と `kabosu-dashboard` の `VEXA_AUTH_COOKIE_NAME` が異なる。
   - ボット名保存キーは現状の `vexa-join-bot-name` を仕様として維持する。通常Dashboardと `kabosu-dashboard` はポートが異なりオリジン分離されるため実害はない。

## 6. セキュリティ・プライバシー

ブランド切り替えは表示設定であり、API認証方式は既存Dashboardと同じ。ユーザートークン、管理APIキー、会議文字起こしデータの保存先は既存実装に従う。

| 項目 | 内容 |
| :-- | :-- |
| 認証方式 | 既存Dashboardと同じ。直接ログイン、OAuth、マジックリンクの設定に依存する |
| URL漏洩時の影響 | Dashboard URLを知った第三者がログイン画面へ到達できる。実データ閲覧可否は認証状態に依存する |
| 不正利用への対策 | API側の認証・レート制限に依存する。Dashboard単体では新しい制限を追加しない |
| ログ | ブランド名変更により新規の個人情報ログは追加しない |

## 7. 外部サービス境界

| サービス | 障害時の挙動 | 検知方法 | 通知先 | 対応者 | 復旧手順 |
| :-- | :-- | :-- | :-- | :-- | :-- |
| API Gateway | 会議一覧、会議参加、文字起こし取得が失敗する | Dashboardのエラー表示、`/api/config` と `/api/vexa/*` の応答確認 | 運用者 | 運用者/開発者 | API Gatewayを復旧し、Dashboardから再試行する |
| Admin API | ログイン、ユーザー管理、APIキー管理が失敗する | ログイン画面の設定エラー表示 | 運用者 | 運用者/開発者 | Admin API設定と管理トークンを確認し、サービスを再起動する |

## 8. 非機能要件

| 項目 | 基準値 | 計測方法 | 前提条件 |
| :-- | :-- | :-- | :-- |
| 既定互換性 | ブランド環境変数未指定でカボス/日本語の既定値テストが通過する | `npm test` | `services/dashboard` で実行 |
| 起動分離 | 通常Compose起動で `kabosu-dashboard` が起動しない | `docker compose config --services` とプロファイル指定時の差分確認 | Compose v2 |
| UI表示 | 主要導線が日本語表示される | ブラウザでログイン、会議一覧、参加モーダルを目視確認 | 日本語限定方針 |

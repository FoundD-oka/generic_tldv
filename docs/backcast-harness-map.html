harness-init は何を、どの順番で処理するか

Issueを受け取ってからPRを作れる状態になるまで、harness-initは「調べる」「計画する」「危なさで分ける」「範囲を囲う」「証拠にする」「改札で止める」を順番に通します。中央の工程表は、各段階の入力、処理、生成物、次に渡すものを並べた機能ロードマップです。

Issue → task-id依頼を追跡単位へ変える
Plan → S/M/L作る前に危なさを決める
Build → Evidence実装結果を証拠へ変換する
Gate → PRready/blockを機械的に判定する
